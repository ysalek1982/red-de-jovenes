# Plan de contenido inicial para piloto

Fecha: 2026-05-18

## Objetivo

Preparar contenido base realista para que Red de Jovenes se sienta lista para un piloto cerrado sin depender de pantallas vacias ni datos improvisados.

## Contenido minimo requerido

| Area | Contenido base | Criterio |
| --- | --- | --- |
| Devocional diario | 7 devocionales activos | Cubrir la primera semana del piloto con reflexiones breves y tono pastoral |
| Mapa mundial | Comunidades base editables | Mostrar directorio real inicial sin contactos personales inventados |
| Juegos de fe | 15 preguntas por juego | Evitar una experiencia demasiado corta |
| Espacio seguro | Normas claras | Explicar respeto, privacidad, reportes y cuidado de jovenes |
| Onboarding | Mensajes de bienvenida | Guiar el primer uso sin lenguaje tecnico |
| Escenarios QA | Recorridos joven/admin/seguridad | Validar que los flujos principales tengan sentido humano |

## Devocionales iniciales

- Deben usar referencias biblicas y reflexiones originales.
- Evitar textos biblicos extensos si no hay version/licencia confirmada.
- Mantener tono juvenil, cristiano, esperanzador y no polemico.
- Incluir una oracion breve final.

## Comunidades iniciales del mapa

- Prioridad Bolivia: Santa Cruz de la Sierra, La Paz y Cochabamba.
- Agregar algunas referencias internacionales para conservar el concepto global.
- Si la comunidad no esta confirmada como real, marcarla como contenido base editable para piloto.
- No usar contactos personales ni enlaces inventados.

## Juegos

- Versiculo Rapido: memoria, identidad, confianza, amor, servicio y sabiduria.
- Trivia Biblica: Evangelios, Antiguo Testamento, Fe y vida.
- Preguntas de dificultad basica/media.
- Explicaciones formativas, sin tono competitivo agresivo.

## Normas y espacio seguro

- Cuidar el lenguaje pastoral.
- Evitar tono policial.
- Explicar que reportar es una herramienta de cuidado comunitario.
- Reforzar privacidad, respeto y acompanamiento.

## Criterios para evitar contenido sensible

- No publicar datos personales.
- No inventar iglesias, lideres o contactos como si fueran confirmados.
- No incluir doctrina controversial como requisito de participacion.
- No usar promesas espirituales manipulativas.
- No citar textos extensos con licencia incierta.

## Pendientes para curaduria humana

- Confirmar comunidades reales antes de abrir piloto amplio.
- Revisar devocionales con un lider pastoral.
- Validar preguntas de juegos con un equipo de contenido.
- Definir protocolo de moderacion para casos sensibles.
