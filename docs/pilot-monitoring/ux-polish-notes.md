# Pulido UX para piloto

## Ajustes aplicados

- Se agrego el acceso “Ayudanos a mejorar” en el menu Mas y en Perfil.
- El formulario de feedback usa lenguaje directo y no tecnico.
- El panel Admin ahora separa feedback, incidentes y reporte de cierre.
- El monitoreo diario muestra cero cuando no hay datos.
- Los incidentes usan severidad y estado claros para operar el piloto.

## Criterios mantenidos

- Seguridad no vuelve al menu visible.
- Reportes y moderacion siguen activos internamente.
- No se agregaron modulos grandes.
- No se usan metricas ficticias.

## Pendiente humano

- Revisar con usuarios reales si “Ayudanos a mejorar” es suficientemente visible.
- Confirmar que el modal no queda tapado por teclado movil.
